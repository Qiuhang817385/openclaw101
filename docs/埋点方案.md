## 四、SEO（元数据 + 结构化数据 + 多语言声明）

以下均可在 `layout.tsx` 中通过取消注释恢复；**完整原始代码**仍在同文件注释块内，或见恢复前 Git 历史。

### 4.1 Next.js `metadata`（Metadata API）

| 字段 | 作用 | 当前状态 |
|------|------|----------|
| `metadataBase` | 站内绝对 URL 解析基准 | **保留** |
| `title` / `description` | 浏览器标签与摘要 | **保留** |
| `icons` | favicon | **保留** |
| `keywords` | `<meta name="keywords">` | **已注释** |
| `robots` / `googleBot` | 索引与预览片段策略 | **已注释** |
| `openGraph` | Facebook/LinkedIn 等分享卡片（含 `og-image.png`） | **已注释** |
| `twitter` | Twitter/X 大图卡片 | **已注释** |
| `verification.google` | Google 站点验证（原值为 `G-86ESEQC7V8`） | **已注释** |
| `alternates.canonical` | 规范 URL | **已注释** |
| `alternates.languages` | `hreflang` 的等价映射（zh / en） | **已注释** |

**说明**：`verification.google` 常规用法是填入 [Google Search Console](https://search.google.com/search-console) 提供的**验证码**（或 HTML 标签中的 `content`），与 GA4 测量 ID `G-xxxx` 并非同一概念；若此前误用 GA ID，恢复 SEO 时建议改为 Search Console 中的正确验证码。

### 4.2 `<head>` 内联标签

| 内容 | 作用 | 当前状态 |
|------|------|----------|
| `application/ld+json` | Schema.org：`@graph` 含 `WebSite`、`Course`、`FAQPage`、`ItemList`（七日教程链接等） | **已注释**（脚本 + 上方 `jsonLd` 常量整块注释） |
| `<link rel="alternate" hrefLang="zh|en|x-default">` | 多语言 SEO，与 URL `openclaw.meetai.fun` / `/en` 对齐 | **已注释** |

子页面若另有 `generateMetadata`，不在本文档范围内，需单独排查。

## 五、小结

| 类别 | 方案 | 用途 |
|------|------|------|
| 统计 | GA4 | 访问与转化分析 |
| 统计 | Plausible | 轻量/隐私向统计 |
| 广告 | AdSense | 广告投放 |
| SEO | Metadata 扩展字段 | 关键词、爬虫规则、OG/Twitter、验证、规范链与语言替代 |
| SEO | JSON-LD | 富结果与知识图谱线索 |
| SEO | hreflang 链接 | 多地区/多语言检索信号 |

当前代码库中未发现组件级自定义事件（如 `gtag('event', ...)`、`plausible('eventName')`）；若有后续交互埋点，建议在本文档追加「事件名 / 触发位置」表。

## 六、启用与关闭

### 6.1 统计与广告脚本

1. 在 `src/app/layout.tsx` 顶部添加：`import Script from 'next/script'`。
2. 取消注释需要的 `<Script>` / `<meta>` 块（AdSense、GA4、Plausible）。

### 6.2 SEO

1. 在 `export const metadata` 对象中，取消包裹 `keywords`～`alternates` 的大段块注释（注意保持对象内逗号语法正确）。
2. 取消 `const jsonLd = { ... }` 整块及其闭合 `*/` 的块注释。
3. 取消输出 JSON-LD 的 `<script type="application/ld+json">` 与三条 `hreflang` `<link>` 的 JSX 注释。

按需分步恢复即可（例如只开 OG、不开统计）。
