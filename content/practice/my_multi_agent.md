## 背景与灵感

一个人做内容生产，往往会遇到这样的困境：

- 想写文案的时候，脑子里全是技术实现的细节
- 写代码的时候，又惦记着今天的选题还没定
- 做运营规划的时候，视觉呈现完全没思路

人的思维在不同专业模式之间切换，需要时间和精力成本。

我的解决方案是：**把不同的专业能力拆分成独立的 AI 智能体，让它们各司其职**。

就像一个公司里有产品经理、开发、设计、运营一样，我的 AI 团队也有明确的分工。用户只需要跟"主会话"对话，剩下的分派和协调工作由系统自动完成。

---

## 整体架构

整个系统采用**主从 + 专业化分工**的模式：

```
用户（主会话）
│
├── 项目管理    → PD（项目规划、任务拆解、进度把控）
├── 工程技术    → 干活专家（全栈开发）、屿航（深度技术研究）
├── 运营/设计/创作 → 三助手协作
└── 平台专项    → 小红书专项助手
```

**核心逻辑**：用户通过主会话下达指令，主会话负责分派任务、追踪进度、汇总结果。用户感知到的只是一个"全能助手"，背后其实是一个完整的团队在协作。

## 团队成员

每个智能体都有自己明确的角色定位和能力边界：

| 成员           | 角色       | 核心职责                     | 特点                       |
| -------------- | ---------- | ---------------------------- | -------------------------- |
| **PD**         | 项目管理   | 项目规划、任务拆解、进度把控 | 统筹全局，擅长拆解复杂任务 |
| **干活专家**   | 全栈工程师 | 前后端代码实现               | 快速开发，稳定输出         |
| **屿航**       | 技术专家   | 深度技术研究、复杂问题攻关   | 攻坚克难，懂原理           |
| **运营助手**   | 运营       | 策略制定、内容分发、数据分析 | 数据驱动，懂平台规则       |
| **设计助手**   | 设计       | UI/UX 设计、视觉产出         | 审美在线，产出快           |
| **创作助手**   | 创作       | 文案、脚本、内容策划         | 懂用户心理，擅长表达       |
| **小红书助手** | 平台专项   | 小红书平台专项运营           | 懂小红书调性               |

### 协作示意

```
当有一个新项目时：

PD（项目规划）
    ↓ 拆解任务
    ├── 干活专家 + 屿航（技术开发）
    ├── 运营助手（制定策略）
    ├── 设计助手（产出视觉）
    └── 创作助手（输出内容）
    ↓ 汇总结果
主会话 → 用户
```

## 任务分派规则

### 基本原则

| 原则         | 说明                                             |
| ------------ | ------------------------------------------------ |
| **复用优先** | 优先派给已有智能体，复用历史上下文，避免重复沟通 |
| **按需创建** | 只有目标不存在时才新建会话                       |
| **精准路由** | 用 sessionKey 精确定位，而非模糊匹配             |

### 任务分流

| 任务类型               | 路由目标   | 说明               |
| ---------------------- | ---------- | ------------------ |
| 项目规划、进度把控     | PD         | 全局统筹           |
| 前后端开发、功能实现   | 干活专家   | 日常开发任务       |
| 复杂技术调研、方案攻坚 | 屿航       | 需要深度研究的问题 |
| 运营策略、内容分发     | 运营助手   | 偏运营侧           |
| 视觉设计、UI 输出      | 设计助手   | 偏设计侧           |
| 文案、脚本、内容策划   | 创作助手   | 偏内容侧           |
| 小红书专项运营         | 小红书助手 | 平台专项           |
| 综合任务               | 多助手协作 | 运营+设计+创作协同 |

## 三助手协作流程

对于设计、创作、运营类的综合任务，三个助手会组成协作链路：

```
运营助手（策略规划）
        ↓
设计助手（视觉设计）
        ↓
创作助手（内容创作）
```

**运作方式**：

1. **运营助手**先分析任务目标，制定策略框架
2. **设计助手**基于策略产出视觉方案
3. **创作助手**在策略和视觉的基础上创作具体内容
4. 主会话汇总三者的输出，形成完整的任务交付

三者通过消息传递串联，用户不需要分别跟每个助手沟通，只需要在主会话下达指令，等待结果即可。

## 设计亮点

| 亮点           | 说明                                                 |
| -------------- | ---------------------------------------------------- |
| **职责清晰**   | 每个智能体专注自己的领域，避免"全能但都不精"的问题   |
| **上下文复用** | 已有会话带历史上下文，每次沟通不需要从零开始         |
| **主从协同**   | 主会话作为协调者，保持全局视图，用户体验统一         |
| **平滑扩展**   | 新增智能体只需注册，无需改动核心逻辑                 |
| **统一入口**   | 所有交互通过 Telegram 完成，用户感知是一个助手在服务 |

### 实际感受

用起来的感觉是：**就像有一个私人团队在为你工作**。

你说"帮我做一个知识库产品"，系统会自动拆解成：

- PD制定项目计划
- 干活专家开始搭建技术框架
- 运营助手规划上线策略
- 创作助手准备推广内容

你只需要确认方向正确，剩下的交给 AI 团队完成。
